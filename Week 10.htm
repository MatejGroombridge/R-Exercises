Exploratory activity

In order to create a package, you will use some tools that:

  • assist you in creating a folder struture where all you code and documentation wil be, organised according to some specific rules
  • build the package; to do so, R takes a folder (the one you just created) filled with some code and documentation and creates the R package. You can think to this process as a very special zip or compress operation, in which the compressing function has expectations on what has to be found in the folder.

The tools will mostly be implemented with console commands. The same operations can be completed clicking on options in the RStudio menu. You can use both, but I will lead you through the details of console commands as that is the easiest way to understand how things work.

Both the folder and the package need names. In this example below, we will use the name cats for both. In general, they can have different names.

The first thing you have to decide is where you want to create the folder containing your code and documentation. Have a look at where you are using the command getwd() and then move to the folder where you want to create your new package, i.e., a new folder named cats. For example, you could want to create a folder names packages and collect all you packages there. Its your decision, spend a couple of minutes deciding how you want to organise your present and future code. From now on I will assume you have create a folder named packages

Create your package directory

After you have moved to the packages folder, its time to create your first package skeleton using the function create():

create("cats")

RStudio will complain and say that it is a bad idea to create a package inside a project. Just ignore it and choose to do it: you will be creating you first package skeleton. In most cases, RStudio will also create a new empty session and move you in the package structure. You can check running getwd() and eyeing the [empty] environment pane. As you are in a new session, you need to reload the devtools and roxygen packages using the commands above.

Right now the package is empty, as you didnt write any code yet. Spend a couple of minutes in the Files pane exploring the structure of the package. You will notice two files, DESCRIPTION and NAMESPACE, and two folders, man (for documentation) and R (for your code). If the man folder is missing, dont worry about that, well create it later. But first, lets build the first package running the build() command from the devtools package (which you previously loaded):

build()

R will do a few checks on the structure of the package, and if everything is OK (which it is right now), it will create a package, in the form of a cats_version.tar.gz file. The file should be available in the packages folder, next to the cats folder. Congratulations! You just created your first [empty] package!

Add functions

We can now create our first function. Create an R script named cat_function.R in your R directory in the package folder and copy the code below.

cat_function <- function(love=TRUE){
    if(love==TRUE){
        print("I love cats!")
    }
    else {
        print("I am not a cool person.")
    }
}

You can now rebuild the package, running again the command

build()

Add documentation

The package roxygen2 makes everything amazing and simple. The way it works is that you add special comments to the beginning of each function, that will later be compiled (processed) into the correct format for package documentation. The details can be found in the roxygen2 documentation I will just provide an example for our cat function.

The comments you need to add at the beginning of the cat function are, for example, as follows:

#' A Cat Function
#'
#' This function allows you to express your love of cats.
#' @param love Do you love cats? Defaults to TRUE.
#' @keywords cats
#' @export
#' @examples
#' cat_function()
 
cat_function <- function(love=TRUE){
    if(love==TRUE){
        print("I love cats!")
    }
    else {
        print("I am not a cool person.")
    }
}

Some prefer creating a new file for each function, others (like myself) create new functions sequentially in one R file, possibly using several R files to cluster the functions according to some criteria. In any case, just make sure to add the documentation comments before each function.

Process your documentation

Now you need to create the documentation. Youve already done the hard work when you wrote those special comments before the function cat_function(). The command document() reads those annotations and does mostly two things: (1) creates the .Rd files in the man directory, (2) adds or replaces the NAMESPACE file in the main directory of the package. For now, dont worry about the NAMESPACE file, its too advanced for this subject. In order to run the document() command you have to be into the package folder. You should be there, but you can check running the usual command getwd().

# Go into the cats folder
getwd("./cats")
document()

If you have configured the Build tools correctly, you can run the same command more easily (i.e. without changing folder) from the RStudio interface by clicking on Build/Document. Feel free to do this. Also remember: everyone forget to compile the documentation before building/installing packages. Try to take a mental note that you need to do this, otherwise you will end up with old documentation in the package, even if you modified it. You can read up more about the files just created in the man directory, and inspect them if you wish, but in terms of steps you need to take, you really dont have to do anything further.

Install

Now it is as simple as installing the package! You need to run this from the parent working directory that contains the cats folder.

# Go back to the root folder
setwd("..")
install("cats")

Now you have a real, live, functioning R package. For example, try typing ?cat_function or cat_function(). You should see the standard help page pop up and the function you wrote running.

It time to head to the DESCRIPTION file and modify it as you prefer. Note that you can decide the version umber of your package. It is good practice to start with a 0.# version, and pass to 1.# when the package is stable and can be safely circulated. Every minor version should advance like 1.0, 1.1, 1.2, , 1.11, 1.12, etc. Every major version of the package advances the leading digit: 2.0, 3.0, etc.

If you want to have even more fun, you can create new functions with their documentation, and repeat the steps just taken.

2. Combine the skills learnt so far - Create an R package

In this exercise, you will solve a similar problem to Exercise 2 in week 9, but this time you will put the code in a package.

You should have in your toolkit from last week the following functions:

  • new_card(name, age, langs=NULL, cols=NULL): a function that creates an object of class card
  • print.card(card): a method to print an object of class card
  • new.cabinet(): a function to create a cabinet, i.e. a list of class cabinet able to contain cards
  • add_card.cabinet(cabinet, card): a function to add a card to a cabinet.
  • list_cards.cabinet(cabinet): a function to list al cards contained in a cabinet
  • del_card.cabinet(cabinet, card_id): a function to delete a card from a cabinet
  • search.cabinet(cabinet, age_min, age_max): a function to search in a cabinet and return a summary of all cards with individuals within a specific age range [age_min, age_max] (boundaries included).

Write an R package named cards that contains all the functions listed above.

As in week 9, we have to teach R about the new methods in town. In the context of an R package, we need to export both the UseMethod statement and the functions (as we already did in the cats package), adding the Roxygen2 comment #' @export before any function, those containing the UseMethod statement and the functions we want users to be able to run. Below you can learn how to export the UseMethod statements and the functions, of which one example is given.

One more detail: please note we have changed search into search_card to avoid conflicts with the base R function search().

# Teach R about the new methods in towm

#' @export
add_card <- function(x, ...){
  UseMethod("add_card")
}

#' @export
list_cards <- function(x, ...){
  UseMethod("list_cards")
}

#' @export
del_card <- function(x, ...){
  UseMethod("del_card")
}

#' @export
search_card <- function(x, ...){
  UseMethod("search_card")
}

#' @export
new_card <- function(name, age, langs=NULL, cols=NULL){
  ## YOUR CODE HERE
}
